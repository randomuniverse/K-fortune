# 변경사항 요약

## 수정 파일 2개:

### 1. server/scheduler.ts (교체)
- `cron-fortune.ts`의 기능을 `scheduler.ts`로 통합
- `--run-now` 플래그로 수동 실행 가능 (cron job 대체)
  → `tsx server/scheduler.ts --run-now`
- p-retry: 운세 생성 실패 시 최대 3회 재시도 (2초~10초 간격)
- p-limit: 동시 처리 최대 2명 (API rate limit 방지)
- 텔레그램 전송도 2회 재시도

### 2. server/fortune-engine.ts (교체)
- 개별 AI API 호출에 p-retry 적용 (각 호출당 2회 재시도)
- 빈 응답도 에러로 처리하여 재시도 트리거
- 실패 시 로그에 어떤 호출이 몇 번째 시도인지 표시

## 삭제 파일 1개:
