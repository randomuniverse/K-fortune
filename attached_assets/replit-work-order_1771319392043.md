## 🎯 문제 진단

현재 Master Archetype의 결과물이 "분석 보고서"처럼 읽히고 스토리텔링 임팩트가 약한 이유:

1. **Few-shot 예시 부재**: 규칙은 많지만 "이런 톤으로 써라"는 구체적 예시가 없어서 GPT가 기본 설명조로 빠짐
2. **교차검증 종합 단계에서 날이 무뎌짐**: 3개 리포트를 합치면서 "2개 이상 공통"이라는 필터가 가장 날카로운 표현을 제거함
3. **Temperature 불균형**: 개별 생성 0.85(창의적) → 종합 0.3(안전) → 최종 결과물이 밋밋해짐
4. **pastInference에서 세 시스템(사주/자미두수/별자리)의 서사적 통합이 약함**: 각 시스템의 키워드를 나열하지 말고, "사주에서 이렇게 나왔는데, 자미두수에서도 같은 이야기를 하고, 별자리까지 같은 방향을 가리킨다"는 수렴형 서사가 필요

#### 1-1. 문체/톤 지시 추가 (프롬프트 최상단, 기존 "당신은 데이터를 단순히 나열하는..." 문단 바로 아래)

```
**[문체 헌법 — 이것이 모든 작성 원칙보다 우선]**

당신은 정장을 입은 영매입니다. 차분하고 단정한 존댓말(~습니다, ~입니다, ~겁니다)을 쓰되, 내용은 칼처럼 날카롭고 소름 끼치게 정확합니다.

문체 규칙:
1. 존댓말 체계: ~습니다/~입니다/~겁니다/~십시오 (단정하고 예의 바른 톤)
2. 짧은 문장과 긴 문장의 리듬 교차. 3문장 이상 같은 길이가 연속되면 안 됨.
3. 핵심 통찰은 한 줄로 끊어서 임팩트를 줌. 예: "이건 드문 일입니다. 아주 드문 일."
4. 추상적 비유 다음에는 반드시 체감 가능한 구체적 현상 묘사가 따라옴.
5. "~할 수 있습니다", "~경향이 있습니다" 같은 약한 표현 금지. "~입니다", "~겁니다"로 단정.
6. 매 섹션의 마지막 문장은 읽는 사람의 가슴에 남는 한 줄로 마무리.
```
