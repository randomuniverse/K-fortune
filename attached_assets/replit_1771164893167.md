# replit.md

## Overview

Korean-language AI fortune-telling web application ("천상의 운세" / "Celestial Fortune") that provides personalized fortune readings through **3 independent Eastern/Western systems cross-validated by a Guardian AI**.

Users register with Telegram ID, birth details (date, time, location), gender, and MBTI, then receive AI-generated fortune readings powered by OpenAI GPT-4o. The app provides three types of fortune analysis — **사주팔자 (Four Pillars/Saju)**, **자미두수 (紫微斗數/Ziwei Doushu)**, and **별자리 (Western Zodiac/Astrology)** — each analyzed independently, then cross-validated through a Guardian Synthesis layer that only adopts content confirmed by 2+ systems.

The app features a mystical/celestial dark theme with gold accents, star field animations, and Korean typography (Cinzel + Quicksand).

## User Preferences

- Preferred communication style: Simple, everyday language.
- Design preference: Apple Human Interface Guidelines style — clean, minimal.

## System Architecture

### Frontend Architecture
- **Framework**: React 18 with TypeScript, bundled by Vite
- **Routing**: Wouter (lightweight client-side router)
  - `/` — Home (landing)
  - `/register` — User registration
  - `/dashboard/:telegramId` — Main dashboard (3 main tabs)
  - `/settings/:telegramId` — User settings
- **State Management**: TanStack React Query for server state (caching, mutations, queries)
- **Forms**: React Hook Form with Zod validation via `@hookform/resolvers`
- **UI Components**: shadcn/ui (new-york style) built on Radix UI primitives, styled with Tailwind CSS
- **Animations**: Framer Motion for page transitions, star field effects, score ring animations, and card animations
- **Styling**: Tailwind CSS with CSS variables for theming. Dark mystical theme with custom colors (deep purple background, gold primary). Fonts: Cinzel (serif, headings) and Quicksand (sans-serif, body)
- **Path aliases**: `@/` → `client/src/`, `@shared/` → `shared/`

### Dashboard Tab Structure
The main dashboard has **3 main tabs**, each with sub-tabs:

1. **오늘의 운세 (Today's Fortune)** — `Sun` icon
   - Daily fortune score card (combined 3-system score)
   - Fortune history list with pagination (6 per page)

2. **2026년 총평 (Yearly Fortune)** — `CalendarDays` icon
   - Sub-tabs:
     - **가디언 총평** — Cross-validated synthesis of all 3 systems
     - **사주 총평** — Saju-only yearly analysis + 12-month calendar
     - **자미두수 총평** — Ziwei-only yearly analysis + 12-month calendar
     - **별자리 총평** — Zodiac-only yearly analysis + 12-month calendar

3. **운명 종합 분석 (Destiny Analysis)** — `Sparkles` icon
   - Sub-tabs:
     - **가디언 리포트** — Deep personality/destiny report (Master Archetype)
     - **사주팔자** — Full Saju chart analysis (pillars, ten gods, special sals, structure patterns, yongshin remedy)
     - **자미두수** — Ziwei Doushu palace/star analysis
     - **별자리 정보** — Western zodiac profile

### Backend Architecture
- **Framework**: Express 5 on Node.js with TypeScript (run via tsx)
- **HTTP Server**: Node.js `http.createServer` wrapping Express
- **API Pattern**: RESTful JSON API under `/api/` prefix. Routes defined in `server/routes.ts`
- **Shared Route Definitions**: `shared/routes.ts` defines API contracts (paths, methods, Zod schemas) shared between client and server
- **AI Integration**: OpenAI SDK (GPT-4o model) configured via Replit AI Integrations environment variables
- **Telegram Integration**: Bot API for sending fortune messages + webhook handler for `/start` command auto-linking
- **Dev/Prod Split**: In development, Vite dev server runs as middleware with HMR. In production, pre-built static files served from `dist/public/`
- **Build Process**: Custom build script (`script/build.ts`) uses Vite for client and esbuild for server, outputting to `dist/`
- **Retry Logic**: `p-retry` library for resilient AI API calls (2 retries, 1.5s–5s timeout)

### Data Storage
- **Database**: PostgreSQL via `DATABASE_URL` environment variable
- **ORM**: Drizzle ORM with `drizzle-zod` for schema-to-validation integration
- **Migrations**: Managed via `drizzle-kit push` (schema push approach)
- **Storage Layer**: `server/storage.ts` implements `IStorage` interface with `DatabaseStorage` class wrapping Drizzle queries

## Fortune Engine Logic (in `server/fortune-engine.ts`)

### 1. Daily Fortune — 3-System Cross-Validation (4 API Calls)

1. **Daily Limit**: 1 fortune per user per day (KST). Returns 429 if already generated.
2. **3-Way Parallel Generation** (3 API calls in parallel):
   - **사주팔자 (Saju)**: Analyzes today's 일진 (daily pillar) against user's 일주 (day pillar), checking 형/충/회/합/원진 relationships and 용신 (yongshin) strength. Returns: score, direction, caution, special, summary.
   - **별자리 (Zodiac)**: Analyzes planetary transits affecting user's sun sign and ruling planet. Returns: score, love, money, health, work, summary.
   - **자미두수 (Ziwei Doushu)**: Analyzes today's 일진 against user's 명궁 (life palace) main stars, 재백궁 (wealth palace), and 부처궁 (spouse palace). Returns: score, summary, advice.
3. **Structured JSON Output**: Each reading validated with Zod schemas.
4. **Guardian Synthesis** (4th API call): Cross-validates all 3 system results:
   - Calculates **coherence score** (0–100) based on agreement level
   - Extracts **common keywords** found in 2+ systems
   - Generates **core message** — the single truth all 3 systems agree on
   - Derives **lucky numbers** from 오행 and 국(Bureau) references
   - Preserves each system's original analysis while harmonizing
5. **Final Score**: Average of 3 system scores, +5 bonus if coherence ≥ 80
6. **Storage**: Saved as both formatted Telegram text and structured FortuneData JSON
7. **Telegram**: Auto-sent to user's linked Telegram chat

### 2. Guardian Report — Deep Destiny Analysis (4 API Calls)

1. **3 Independent AI Analyses** (parallel, temperature 0.85):
   - Same comprehensive prompt with full saju chart, personality, ziwei palaces/stars, and zodiac data
   - Each generates: coreEnergy (archetype title), pastInference (400+ chars), currentState, bottleneck, solution, businessAdvice
   - Prompt enforces minimum character counts, no-name policy ("당신" only), and "Step 1 (inner conflict) → Step 2 (behavioral patterns)" structure
2. **Keyword Voting**: Keywords appearing in 2+ reports are kept as common keywords
3. **Guardian Synthesis** (4th call, temperature 0.3): Merges 3 reports, only adopts 2/3+ consensus content, expands depth
4. **Result**: Stored in `guardianReports` table. Currently: businessAdvice active, loveAdvice/healthAdvice = null (Coming Soon)

### 3. Yearly Fortune — 3-System Independent Analysis + Guardian Synthesis (4 API Calls)

1. **3 System-Specific Prompts** (parallel, temperature 0.85–0.9):
   - **사주팔자**: Analyzes 세운 (yearly pillar) vs 일주, 대운 interaction, 용신 strength, 십성 flow. Returns: summary, keywords, monthlyFlow (12 months)
   - **자미두수**: Analyzes 유년 (yearly flow) through palace stars — 명궁, 재백궁, 관록궁, 부처궁, 질액궁, 천이궁. Returns: summary, keywords, monthlyFlow
   - **별자리**: Analyzes major planetary transits (Jupiter, Saturn, Uranus), ruling planet position, retrograde periods. Returns: summary, keywords, monthlyFlow
   - Each system has strict tone rules: balanced analyst, no hype, always pair opportunities with risks
2. **Keyword Cross-Check**: Keywords appearing in 2+ systems are extracted
3. **Monthly Score Averaging**: Each month's score = average across 3 systems
4. **Guardian Synthesis** (4th call, temperature 0.35):
   - Generates: overallSummary (1000+ chars), businessFortune, loveFortune, healthFortune (each 500+ chars), coherenceScore, combined monthlyFlow
   - Monthly summaries start with "O월은 OO(간지)의 기운입니다."
5. **Storage**: Per-system data stored separately (sajuMonthlyFlow, sajuSummary, ziweiMonthlyFlow, ziweiSummary, zodiacMonthlyFlow, zodiacSummary) alongside guardian synthesis in `yearlyFortunes` table
6. **Caching**: Yearly/Guardian reports are stored and reused. Regeneration requires explicit `regenerate: true` flag.

## Saju Engine (in `shared/saju.ts` — 1,252 lines)

Pure TypeScript implementation of Korean Four Pillars astrology (no AI dependency):

### Core Calculations
- **Year/Month/Day/Hour Pillar (年/月/日/時柱)**: Heavenly Stems (천간) + Earthly Branches (지지) calculation
- **Month Pillar**: Based on 절기 (solar terms) approximate dates, not calendar months
- **Ten Gods (십성)**: Calculated from day stem relationships (비견, 겁재, 식신, 상관, 편재, 정재, 편관, 정관, 편인, 정인)
- **Branch Ten Gods**: Hidden stems (지장간) with weighted analysis
- **Five Element Ratios (오행 분포)**: Weighted calculation including hidden stems (지장간 가중치)
- **Day Master Strength (일간 강약)**: 극왕/왕/중화/약/극약 based on seasonal energy + support ratio
- **Yongshin (용신)**: Protective element determination based on day master strength and five element balance
- **Daeun (대운)**: 10-year major luck cycles with forward/reverse calculation based on gender + year stem polarity
- **Chinese Zodiac Animal**: 띠 (rat through pig)

### Special Characteristics Detection
- **괴강살 (魁罡殺)**: Extreme decisiveness — checks for specific day pillar combinations (庚辰, 庚戌, 壬辰, 壬戌)
- **도화살 (桃花殺)**: Star charisma — checks year/day branch against 인오술→묘, 신자진→유, etc.
- **화개살 (華蓋殺)**: Artistic solitude — checks for 인오술→술, 신자진→진, etc.
- **역마살 (驛馬殺)**: Travel/movement energy — checks for branch migration patterns

### Structure Patterns
- **식상생재 (食傷生財)**: Creative energy → wealth generation (식상 + 재성 present)
- **관인상생 (官印相生)**: Authority + scholarship synergy (관성 + 인성 present)
- **상관견관 (傷官見官)**: Rebel innovator (상관 + 정관 collision)
- **재다신약 (財多身弱)**: Opportunity overload (재성 many + weak day master)

### Personality Analysis (`analyzeSajuPersonality`)
- Day stem personality mapping (10 types: 갑=big tree through 계=dew)
- Ten god profile narrative
- Talent mapping by dominant element
- Yongshin remedy (개운법): lucky time, place, direction, color, activity, food per element
- Weakness analysis (considers 괴강살 + 신약 special combination)

### Yearly/Monthly Fortune Calculation
- `calculateYearlyFortune(chart, year)`: Stem/branch relationship analysis against day pillar
- `calculateMonthlyFortunes(chart, year)`: 12-month pillar analysis with scores and keywords

## Ziwei Doushu Engine (in `shared/ziwei.ts` — 279 lines)

Purple Star Astrology (紫微斗數) implementation:

- **Bureau (局) Calculation**: 수2국/목3국/금4국/토5국/화6국 — determines life rhythm
- **Life Palace (命宮) Determination**: Based on birth month and hour
- **14 Major Stars**: 자미(Emperor), 천기(Strategist), 태양(Sun), 무곡(General), 천동(Child), 염정(Passionate), 천부(Empress), 태음(Moon), 탐랑(Desire), 거문(Scholar), 천상(Minister), 천량(Sage), 칠살(Warrior), 파군(Destroyer)
- **Palace Star Distribution**: Stars placed in 명궁 (life), 재백궁 (wealth), 부처궁 (spouse), 관록궁 (career), 천이궁 (travel), 질액궁 (health)
- **Each Star Profile**: name, keyword, personality, loveStyle, wealthStyle, socialStyle
- **Interpretation Generation**: Comprehensive narrative combining palace placements

## Development Notes

- **Time Zone**: All date logic uses KST (UTC+9) — `getKoreaTime()` helper in fortune-engine
- **Daily Pillar Calculation**: Uses Julian Day Number (JDN) for accurate 일진 stem/branch
- **Zod Validation**: All AI responses parsed through Zod schemas; failures trigger retries or fallbacks
- **Error Handling**: Fortune generation has fallback data (score 50, "분석 대기 중") on complete failure
- **Telegram Webhook**: `/start` command auto-links user by matching telegramId, handle, or chatId
- **Replit Plugins**: `@replit/vite-plugin-runtime-error-modal`, `@replit/vite-plugin-cartographer`, `@replit/vite-plugin-dev-banner` (dev-only)
